About My HTTP Server

This website is served by a custom lightweight HTTP server written in Java. The server handles HTTP requests, parses request lines and headers, routes endpoints, and generates responses without using any web framework.

Features

  • Custom HTTP request parser
  • Routing system for endpoints
  • Special endpoints like /httpbin, /video/*, /static/*
  • Static file serving
  • Multithreaded request handling

Explaining Special Endpoints

There are three variations of special endpoints. These include /httpbin/stream/n (the max the server can handle seems to be about ~30), /video/vim, /video/brunson, /static/index.html, /static/about.html. The /httpbin/stream/n endpoint sends a proxy request to https://httpbin.org/, and the server responds with the data via chunked transfer encoding. The /video endpoints sends a video back to the client, they are read in one chunk fully in memory. The /static endpoints are similar, they're static html files which are also read in memory and sent. Both the videos and files are stored in the server.

Design

The design of this server was to meant to emulate the sort of web servers and frameworks such as NGINX or Spring Boot. For example, NGINX uses a predefined config file called mime.types which maps a files extension to its corresponding mime type. I implemented this as a static field in my StaticFileEndpoint, which utilizes a map data structure to determine the mime type. There are 3 main components for interpreting and responding to a request, it goes through the Server -> Response Handler -> Response Writer. The server receives the connection and delegates request parsing to a request parser component. It then delegates to the response handler which handles how to interpret the request and deals with special endpoints, and this uses the response writer which writes the actual bytes to the outputstream. The special endpoints are stored in a Special Endpoint Registry class that's configured during start up of the server. The user defines the router map which allows you to customize how certain endpoints are interpreted, as you can see I specified the /yourproblem and /myproblem endpoints to return 400 and 500 responses, respectively.

How to Run

To run via docker use these commands:

  • mvn clean package
  • docker build -t imagename:tag .
  • docker run -p 5000:5000 imagename:tag

To run via maven use these commands:

  • mvn clean package
  • java -jar target/httpserver-1.0-SNAPSHOT.jar
